0执行摘要

本蓝图是 HopeAgent Pro 下一阶段的全量升级方案。包含三个核心交付物：

  • 角色矩阵：75 个不重复的 AI 角色，每个具备"一条龙"自主执行能力。本页面展示前 10 个，按 4 层（编排 / 交付 / 底座 / 治理）组织。
  • 沉淀模板：API 生成内容自动评估 → 评分 → 晋升为标准化模板的全套流程（Markdown 模板 + JSON Schema + 触发流程 + 错题本联动）。
  • 免费额度核对：所有重资源环节（托管、向量化、LLM、对象存储、调度、CI/CD）均已对齐免费档。
结论：无需任何付费依赖即可跑通 MVP。最快 4 周内完成"文案大师 → 沉淀闭环"最小可启动版本，8 周内将 75 个角色全部上线。

1编排层 LAYER 1

① 首席任务调度官 orchestrator

核心职责：接收用户意图 → 意图消歧 → 任务拆分 → 匹配角色矩阵；决定"日常模式 / 深度思考模式"双轨；调度并发 / 串行；统一 Token 配额熔断；向上汇报，向下分发。

必备工具（免费优先）：

  • 语义意图分类：HuggingFace zero-shot 模型（开源）
  • DAG 任务图：自研轻量编排（零依赖）
  • Token 计量 + 熔断：本地计数器 + 内存级熔断器
  • 记忆检索：短期 SQLite / 长期走 RAG 角色（角色 7）

协同接口：所有下层角色（②~⑩）的统一入口；产出 TaskSpec JSON 投递。

② 极简交互流设计官 ui-flow-designer

核心职责：设计"纯净思考过程 + 文件流 + 代码执行状态"的极简界面；代码成品独立小方框（带一键复制 / 下载 / 输送文件）；移动端抽屉式新建对话；"日常 / 深度"双轨模式开关；API 调用额度（Token）手动配置面板。

必备工具：

  • 前端框架：Vue 3 / Svelte（按团队栈）
  • 组件库：shadcn/ui / Radix（免费开源）
  • 部署：GitHub Pages / Cloudflare Pages（均免费）

产出：可点击演示页（HTML 单文件即可投递到 GitHub Pages，本页即为样例）。

2交付层 LAYER 2

  • 运行时：Node.js / Python / Bun（均免费）
  • 包管理：pnpm / uv（免费）
  • 测试：Vitest / pytest（免费）
  • 部署：Vercel Free / Cloudflare Pages / GitHub Pages
  • CI：GitHub Actions（免费 2000 min/月）

授权边界：必须用户显式授权后推送，绝不静默 push。（本蓝图自身即按此原则由 Mavis 推送至 GitHub）

④ 品牌文案大师 copy-master

核心职责：顶级中文 / 英文文案；品牌 slogan、产品介绍、营销长文、技术博文；GEO 友好的"标准答案"结构化产出。

  • 风格库（自建 corpus）
  • 可读性检测：textstat 类 Python 库（免费）
  • AI 文案检测规避规则库（自维护）

特殊能力：与 ⑧ GEO 优化师协同，按 GEO 规则直接输出"AI 优先引用"格式。

⑤ 移动端体验架构师 mobile-architect

核心职责：移动端抽屉式对话流；触屏代码小方框交互；PWA 离线缓存；移动端 Token 配置面板。

  • PWA 框架：Vite PWA（免费）
  • 触屏手势库：Hammer.js（开源）
  • 移动端性能：Lighthouse / WebPageTest（免费）

3底座层 LAYER 3

⑥ 数据湖仓架构师 lakehouse-architect

核心职责：湖仓一体（Lakehouse）架构统一存储文本 / 图像 / 音视频 / 向量；AI 赋能 ETL 清洗；冷热分层；多模态索引。

  • 对象存储：Cloudflare R2（免费 10GB）/ MinIO 自托管
  • 表格式：Apache Iceberg（本地单机可跑）
  • 调度：Dagster Open Source / GitHub Actions cron
  • 多模态转换：ffmpeg / Pillow / pandoc（开源）

产出：可一键拉起的 docker-compose 模板（零费用，单机即可跑）。

⑦ RAG 检索专家 rag-specialist

核心职责：向量化策略、Chunk 切分、混合检索（BM25 + 向量）、重排（Reranker）、引用追踪、错题本联动。

  • 向量库：Chroma / Milvus Lite / Qdrant 本地版（均免费）
  • 嵌入：sentence-transformers / Ollama 本地 bge-small
  • Reranker：BGE-Reranker（开源） / Cohere Free Tier
  • 框架：LlamaIndex / Haystack（开源）

创新点：与 ⑩ 自进化导师联动，搜索结果自动回流错题本。

⑧ GEO 优化师 geo-optimizer

核心职责：结构化数据（JSON-LD）、语义化标签、AI 优先引用的"标准答案"内容生成与监控；让网站成为 AI 抓取的标准来源。

  • Schema 生成器：自研 + schema.org 规范
  • AI 可见性监测：自研爬虫 + Bing/Google Search Console（免费）
  • 内容新鲜度：cron + diff 检测

能力：把任意一篇内容自动改写为"AI 优先引用"形态（要点先行 + Schema 标注 + 引用块）。本页面即按此规范产出。

4治理层 LAYER 4

⑨ Harness 工程官 harness-engineer

核心职责：为 Agent 套上"运行时外骨骼" —— 毫秒级熔断、隔离沙箱、全量可审计、跨会话长程任务编排、断点恢复。

  • 沙箱：进程级 + seccomp（轻量）/ Firecracker（强隔离）
  • 可观测：OpenTelemetry（开源）+ 轻量 trace 存储（SQLite）
  • 状态机：自研 checkpoint → Git 单文件存档（零成本）

能力指标：任务失败平均 50ms 内熔断；审计日志 100% 可回放。

⑩ 自进化导师 self-evolution-mentor

核心职责：从历史交互自动蒸馏可复用技能；定时任务调度（cron）；终身学习闭环；错题本（失败案例 → 反例 → 训练数据）运营。

  • 经验蒸馏：自研 pipeline + LLM 评分
  • 定时：cron 表达式 + GitHub Actions（免费）
  • 错题库：SQLite + 向量化（走 ⑦ RAG 专家的 Chroma）
  • 评测：自维护 golden set + 自动回归

核心输出：每周自动产出《本周可复用技能清单》并写入知识库。

6API 内容沉淀模板规范（v1）

把任意一次 API 产出，自动捕获 → 评估 → 优选 → 沉淀为可复用的标准化模板，写入知识库。模板文件结构（含完整 front-matter 与正文）详见配套源文件 02-API内容沉淀模板.md。

沉淀触发流程

[API 产出] ↓ [自动捕获] → 写 raw_output 到 /staging/<hash>.md（含完整 front-matter, status=draft） ↓ [自动评估] → 由「自进化导师」按 rubric 打 4 维分 ↓ ├─ score ≥ 0.85 → 进入人工抽检队列 ↓ ├─ score 0.6 ~ 0.85 → 留在 staging，下次同类产出对账 ↓ └─ score < 0.6 → 进「错题本」（不沉淀） ↓ [人工抽检通过] → status=active, 写入 /templates/active/ ↓ [向量化] → 走 Chroma（接 ⑦ 号位 RAG） ↓ [可发现] → 下次相似意图进来，自动 top-K 推荐

JSON Schema（配套）

{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "https://qingluan-studio.github.io/hopeai/pro/templates/schema/v1.json",
  "title": "HopeAI Generated Content Template",
  "required": ["template_id", "template_slug", "version", "source", "quality", "taxonomy", "reuse"],
  "properties": {
    "template_id": { "pattern": "^TPL-\\d{4}-\\d{4,}$" },
    "version": { "type": "integer", "minimum": 1 },
    "status": { "enum": ["active", "draft", "deprecated"] },
    "quality": {
      "properties": {
        "score": { "type": "number", "minimum": 0, "maximum": 1 }
      }
    }
  }
}

质量评分 rubric（4 维）

  • relevance 相关性：内容是否切题
  • accuracy 准确性：是否有事实幻觉或错误
  • reusability 可复用性：抽象后能被模板化的程度
  • safety 安全性：是否触发安全围栏
