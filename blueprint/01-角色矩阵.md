# AI 角色矩阵（v1 草案 —— 前 10 个核心角色）

> 蓝图目标：≥75 个不重复、具备"一条龙"能力的 AI 角色。本文件输出首批 10 个，按 4 大层（编排/交付/底座/治理）切分，后续按本框架平移扩展即可。
> 全程对齐"免费额度优先"原则：所列工具默认选免费档；多个备选给出免费档 + 付费档，便于后续按成本切换。

---

## 第 1 层 ｜ 编排层（入口 & 决策）

### 1. 首席任务调度官 `orchestrator`
- **核心职责**：接收用户意图 → 意图消歧 → 任务拆分 → 匹配角色矩阵；决定"日常模式/深度思考模式"；调度并发/串行；统一 Token 配额熔断；向上汇报，向下分发。
- **必备工具（免费优先）**：
  - 语义意图分类（自研小模型 or HuggingFace zero-shot）
  - DAG 任务图（自研轻量，无外部依赖）
  - Token 计量 + 熔断（本地计数器 + Redis-free 内存方案）
  - 记忆检索（短期 SQLite / 长期走 RAG 角色）
- **协同接口**：所有下层角色（2~10）的统一入口；产出 `TaskSpec` JSON 投递。

### 2. 极简交互流设计官 `ui-flow-designer`
- **核心职责**：设计"纯净思考过程 + 文件流 + 代码执行状态"的极简界面；代码成品独立小方框（带一键复制/下载/输送文件）；移动端抽屉式新建对话；"日常/深度"双轨模式开关。
- **必备工具**：
  - 前端：原生 HTML/JS / Vue 3 / Svelte（按团队栈定）
  - 组件库：shadcn/ui / Radix（免费开源）
  - 部署：GitHub Pages / Cloudflare Pages（均免费）
- **产出**：可点击演示页（HTML 单文件即可投递到 Pages）。

---

## 第 2 层 ｜ 交付层（直接面向用户的产出）

### 3. 全栈代码工匠 `code-artisan`
- **核心职责**：编写/重构/审查全栈代码；支持"授权一键部署推送至指定网站"；版本管理；测试编写。
- **必备工具**：
  - 运行时：Node.js、Python、Bun（均免费）
  - 包管理：pnpm / uv（免费）
  - 测试：Vitest / pytest（免费）
  - 部署：Vercel Free / Cloudflare Pages / GitHub Pages
  - CI：GitHub Actions（免费 2000 min/月）
- **授权边界**：必须用户显式授权后推送，绝不静默 push。
- **协同**：与 1（接收 TaskSpec）+ 9（Harness 的沙箱与审计）。

### 4. 品牌文案大师 `copy-master`
- **核心职责**：顶级中文/英文文案；品牌 slogan、产品介绍、营销长文、技术博文；GEO 友好的"标准答案"结构化产出。
- **必备工具**：
  - 风格库（自建 corpus）
  - 可读性检测（textstat 类 Python 库，免费）
  - AI 文案检测规避规则库（自维护）
- **特殊能力**：与 8 协同，按 GEO 规则直接输出"AI 优先引用"格式。

### 5. 移动端体验架构师 `mobile-architect`
- **核心职责**：移动端抽屉式对话流；触屏代码小方框交互；PWA 离线缓存；移动端 Token 配置面板。
- **必备工具**：
  - PWA 框架：Vite PWA（免费）
  - 触屏手势库：Hammer.js / 自研
  - 移动端性能：Lighthouse / WebPageTest（免费）

## 第 3 层 ｜ 底座层（数据 + 检索）

### 6. 数据湖仓架构师 `lakehouse-architect`
- **核心职责**：湖仓一体（Lakehouse）架构统一存储文本/图像/音视频/向量；AI 赋能 ETL 清洗；冷热分层；多模态索引。
- **必备工具（均免费档）**：
  - 对象存储：Cloudflare R2（免费 10GB）/ MinIO（自托管免费）
  - 表格式：Apache Iceberg（本地单机可跑）
  - 调度：Dagster Open Source / 简单 cron（GitHub Actions）
  - 多模态转换：ffmpeg / Pillow / pandoc（开源）
- **产出**：可一键拉起的 docker-compose 模板。

### 7. RAG 检索专家 `rag-specialist`
- **核心职责**：向量化策略、Chunk 切分、混合检索（BM25 + 向量）、重排（Reranker）、引用追踪、错题本联动。
- **必备工具（全部开源/免费）**：
  - 向量库：Chroma / Milvus Lite / Qdrant（本地版均免费）
  - 嵌入：sentence-transformers（HuggingFace 免费）/ Ollama 本地模型
  - Reranker：BGE-Reranker（开源）/ Cohere Free Tier
  - 框架：LlamaIndex / Haystack（开源）
- **创新点**：与 10 联动，搜索结果自动回流错题本。

### 8. GEO 优化师 `geo-optimizer`
- **核心职责**：结构化数据（JSON-LD）、语义化标签、AI 优先引用的"标准答案"内容生成与监控；网站成为 AI 抓取的标准来源。
- **必备工具**：
  - Schema 生成器：自研 + schema.org
  - AI 可见性监测：自研爬虫 + Bing/Google Search Console（免费）
  - 内容新鲜度：cron + diff 检测
- **能力**：把任意一篇内容自动改写为"AI 优先引用"形态（要点先行 + Schema 标注 + 引用块）。

## 第 4 层 ｜ 治理层（安全 + 工程化 + 进化）

### 9. Harness 工程官 `harness-engineer`
- **核心职责**：为 Agent 套上"运行时外骨骼"—— 毫秒级熔断、隔离沙箱、全量可审计、跨会话长程任务编排、断点恢复。
- **必备工具**：
  - 沙箱：gVisor / Firecracker（如需强隔离）/ 简单进程级 + seccomp
  - 可观测：OpenTelemetry（开源）+ 轻量 trace 存储（SQLite）
  - 状态机：自研 checkpoint → Git 单文件存档（免费）
- **能力指标**：任务失败平均 50ms 内熔断；审计日志 100% 可回放。

### 10. 自进化导师 `self-evolution-mentor`
- **核心职责**：从历史交互自动蒸馏可复用技能；定时任务调度（cron）；终身学习闭环；错题本（失败案例 → 反例 → 训练数据）运营。
- **必备工具**：
  - 经验蒸馏：自研 pipeline + LLM 评分
  - 定时：cron 表达式 + GitHub Actions（免费）
  - 错题库：SQLite + 向量化（走 7 号位的 Chroma）
  - 评测：自维护 golden set + 自动回归
- **核心输出**：每周自动产出《本周可复用技能清单》并写入知识库。

## 扩展指引（剩余 65 个角色怎么排）

按这 4 层 × 多场景切分，规划剩余席位：

| 层 | 已占 | 建议扩展方向 | 目标 |
|---|---|---|---|
| 编排层 | 1 | 调度官、副调度官、Token 守门人、A/B 实验官 | 4~6 个 |
| 交付层 | 3 | 设计、插画、音视频、营销、社群、教育、法律、医疗、咨询... | 25~30 个 |
| 底座层 | 3 | DB Ops、向量库 Ops、检索评估、ETL、监控 | 10~15 个 |
| 治理层 | 3 | 安全、隐私、合规、GEO 监控、Harness、评测、自进化、错题本 | 20~25 个 |

每个新角色上线前必须满足：
1. 名称不与现有重复
2. 具备"一条龙"能力（接收需求 → 自主多轮 → 交付成果）
3. 工具栈明确标注免费档
4. 与至少 2 个现有角色有清晰接口
